    Interface ArgumentConfig

    Configuration for a command argument extracted from a pattern.

    This interface represents the parsed configuration of an argument placeholder from a command pattern. It's extracted by analyzing the pattern string and defines how the argument should be parsed and validated.

    Example

    Pattern "get item:object@room" produces:

    {
      name: "item",
      type: ArgumentType.OBJECT,
      required: true,
      source: "room"
    }
    interface ArgumentConfig {
        name: string;

        type: ArgumentType;

    For OBJECT type arguments, specifies where to search for the object. Extracted from the @ modifier in the pattern (e.g., "item:object@inventory"). - "room": Search only in the current room's contents - "inventory": Search only in the actor's inventory - "all": Search both room and inventory (default)
